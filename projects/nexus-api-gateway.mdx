Nexus is a production-grade API gateway built to handle the demands of distributed microservice architectures. It centralizes request routing, enforces rate limits, validates JWT tokens, and streams live traffic analytics to a real-time dashboard — all in a single, composable layer that sits in front of your services.

## Overview

Nexus was designed for throughput. It currently serves **5M+ daily requests** across distributed microservices, with sub-millisecond overhead on the hot path. Rather than bolt authentication and observability onto individual services, Nexus treats these as first-class gateway concerns — keeping downstream services lean and consistent.

The architecture leans on Redis for fast, atomic rate limiting state and WebSockets for pushing live traffic telemetry to the analytics dashboard without polling.

## How it works

The rate limiter uses a Redis pipeline to perform an atomic increment and TTL set in a single round trip. This keeps the overhead minimal even at high request volumes:

```js
// Redis-based sliding window rate limiter
// A single INCR + EXPIRE pipeline is sub-millisecond
const pipeline = redis.pipeline();
pipeline.incr(`rate:${req.ip}`);
pipeline.expire(`rate:${req.ip}`, WINDOW_SECONDS);
const [count] = await pipeline.exec();
if (count > LIMIT) return res.status(429).json({ error: 'Rate limit exceeded' });
```

Each request that passes rate limiting is then routed to the appropriate upstream service. JWT validation happens synchronously on the same request cycle — the decoded payload is forwarded as a request header so downstream services can trust the identity context without re-verifying the token.
