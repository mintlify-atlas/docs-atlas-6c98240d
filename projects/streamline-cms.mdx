Streamline is a headless content management platform built for teams that need flexibility without the overhead of traditional CMS setups. It supports unlimited tenants, ships with a clean Next.js admin panel, and delivers content changes downstream via configurable webhook pipelines — making it a strong foundation for content-heavy production environments.

## Overview

Unlike monolithic CMS platforms, Streamline separates the content layer entirely from the presentation layer. Content teams manage everything through the admin UI; developers consume a clean API from whatever frontend they choose. The multi-tenant model means a single Streamline deployment can power multiple independent content organizations, each with its own data isolation, roles, and webhook configurations.

Streamline currently powers several production content teams, with a data model built on PostgreSQL and Prisma for type-safe schema management.

## Webhook payload

When content is published, Streamline fires a signed POST request to all configured webhook endpoints for that tenant. Here's an example payload:

```json
{
  "event": "content.published",
  "tenant": "acme-corp",
  "content": {
    "id": "post-123",
    "type": "article",
    "status": "published"
  }
}
```

The `tenant` field scopes the event so multi-tenant webhook consumers can filter reliably. Consumers can verify the origin and react to the event in any way they choose — CDN invalidation, notification services, or custom integrations.
